

CARDINALITA
OSPITE (0, N) - (0, N) EVENTO => ad ogni evento possono partecipare da 0 a N ospiti e ogni ospite può
partecipare a 0 o N eventi.

EVENTO (0, N) - (0, N) CLIENTE => non tutti i clienti sono avvisati degli eventi (solo quelli iscritti alla 
newsletter) e i clienti avvisati possono essere avvisati di 0 o N eventi.

CLIENTE (1, N) - (1, 1) PRENOTAZIONE_OMBRELLONE => una prenotazione ombrellone può essere fatta da solo un cliente, mentre un cliente può fare da 1 a N 
prenotazioni (non parto da 0 perche per essere registrato come cliente significa che ha fatto almeno una prenotazione)

CLIENTE (1, N) - (1, 1) PRENOTAZIONE_TAVOLO => una prenotazione tavolo può essere fatta da solo un cliente, mentre un cliente può fare da 1 a N 
prenotazioni

CLIENTE (0, N) - (1, 1) DOCUMENTO => ogni cliente può avere oppure no un documento, mentre un documento si riferisce ad un solo cliente

PRENOTAZIONE_OMBRELLONE (1, 1) -  (1, N) OMBRELLONE => una prenotazione_ombrellone può essere fatta per pronotare da 1 ombrellone
e se prenoto un ombrellone, esso può far parte anche di più prenotazioni (per periodi di tempo diversi)

PRENOTAZIONE_TAVOLO (1, 1) - (1, N) TAVOLO => una prenotazione_tavolo può essere fatta per pronotare da 1 a N tavoli
mentre se prenoto un tavolo esso può comunque far parte di un'altra prenotazione (una per il pranzo e una per la cena)

TAVOLO (1, N) - (1, 1) ORDINE => un tavolo può effettuare da 1 a N ordini ma un ordine appartiene solo ad un tavolo
ovvero 2 tavoli diversi non possono effettuare lo stesso ordine (magari hanno stessi menu scelti ma hanno id diverso)

ORDINE (0, N) - (0,N) MENU => in ogni ordine vi possono essere da 0 a N MENU, mentre ogni MENU può essere presente in 0 o N ordini
(ho messo 0 perché un menu potrebbe anche non essere mai ordinato)

ORDINE (0, N) - (0,N) PIATTO => ogni ordine può contenere da 0 a N piatti, mentre ogni piatto può essere contenuto da 0 a N ordini

MENU (1, N) - (2, N) PIATTO => un piatto può appartenere a più menu, e un menu contiene
da 2 a N piatti

OMBRELLONE (1, 1) - (1, N) RIGA => ad ogni riga appartengono da 1 a N ombrelloni, e ogni
ombrellone appartiene ad 1 riga
 

RIGA (1, N) - (1, 1) PREZZO => ogni riga ha da 1 a N prezzi (in base al periodo), mentre
ogni prezzo appartiene ad una sola riga (qui dubbio perchè in teoria dalla 3 fila in poi
più righe hanno lo stesso prezzo)
